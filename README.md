# AI Paper Assistant New

面向硕博生和科研新手的学术 Idea Agent。项目目标不是单纯复刻一个论文搜索页，而是做一个从“研究方向”到“可讨论研究提案”的最小可行产品。

用户输入领域、研究兴趣、已有数据或方法偏好后，系统通过论文检索、AI 总结、研究空白识别、新颖性验证、方法缺口分析和提案生成，帮助用户快速形成可以拿去和导师讨论的论文 idea。

## 1. 产品定位

### 目标用户

- 硕士生、博士生、科研助理
- 正在选题、开题、写 proposal 或找研究创新点的人
- 对某个领域有初步兴趣，但还不确定研究空白、理论框架、数据和方法是否可行的人

### 核心价值

- 把传统数周的文献摸索压缩为一套可复用的 Agent 工作流
- 让用户不是只“搜论文”，而是完成“找 idea、验 idea、写提案”的闭环
- 输出必须带文献依据，避免纯 AI 幻觉式建议

## 2. MVP 功能范围

MVP 聚焦一个完整工作流：**从研究方向到 300 字研究提案草稿**。

### 2.1 研究空白扫描

用户输入：

- 学科领域
- 具体研究方向
- 用户身份，例如硕士生、博士生、科研新人
- 时间范围，默认近 3 年

系统输出：

- 过去 3 年重要论文 10 篇
- 每篇论文的核心贡献
- 当前仍未解决的 3 个主要研究挑战
- 基于挑战生成 2-3 个论文 idea

实现要求：

- 检索结果必须展示标题、作者、年份、来源、摘要、链接或 DOI
- AI 总结必须引用对应论文
- 不直接承诺“绝对创新”，只给出“可能的研究空白”

### 2.2 对比式提问筛选 Idea

用户拿到初步 idea 后，可以继续追问：

- 该议题最主流的 3 个理论框架是什么？
- 每个理论框架适合解释什么问题？
- 该 idea 是新理论、新场景、新数据，还是新方法？
- 对普通硕博生而言，哪一个 idea 更可行？

系统输出：

- 理论框架对比表
- 每个 idea 的可行性评分
- 推荐优先推进的 1-2 个方向

### 2.3 Idea 新颖性验证

用户输入某个 idea 或关键词，系统自动生成检索式并执行验证。

验证方式：

- 搜索 idea 关键词
- 搜索关键词 + `review`
- 限制年份为 2025 和 2026，查看最新进展
- 检查是否已有高度相似研究或综述

系统输出：

- 已有相关论文列表
- 是否已有综述
- 方向成熟度判断：新兴、发展中、成熟、过度拥挤
- 新颖性风险说明
- 可调整的切入角度

### 2.4 数据可行性分析

用户输入：

- 可用数据集描述
- 样本来源、样本量、时间范围
- 变量列表
- 数据获取方式

系统分析：

- 样本覆盖是否匹配研究问题
- 变量是否足够支撑分析
- 预期结论是否可证伪
- 是否存在明显偏差或伦理风险
- 数据范围能反推出哪些研究问题

MVP 阶段先支持粘贴文本形式的数据说明，不要求上传和解析真实数据文件。

### 2.5 方法缺口分析

系统回答：

- 该议题过去主要使用哪些方法
- 哪种方法使用最多
- 这些方法的局限是什么
- 是否有新兴方法尚未广泛引入
- 引入新方法后能回答哪些过去答不了的问题

重点方法类型：

- 内容分析
- 问卷调查
- 实验法
- 访谈
- 计算社会科学
- 机器学习文本分析
- 因果推断
- 网络分析

### 2.6 研究提案草稿生成

用户完成前五步后，系统生成约 300 字研究提案。

提案包含：

- 研究背景
- 研究问题
- 文献空白
- 理论或方法创新
- 数据来源
- 研究方法
- 预期贡献

输出要求：

- 适合直接拿去和导师讨论
- 同时提供“更学术版”和“口语汇报版”
- 附带关键参考文献列表

## 3. 页面与交互设计

### 3.1 主页面布局

- 左侧栏：项目历史、我的文献库、设置、登录入口
- 中间区：Agent 对话和工作流步骤
- 右侧栏：证据面板，展示论文、引用、检索式、筛选条件

### 3.2 工作流步骤

建议用 6 步进度条或步骤卡片：

1. 研究空白扫描
2. 理论框架筛选
3. 新颖性验证
4. 数据可行性分析
5. 方法缺口分析
6. 提案生成

每一步都应支持：

- 继续追问
- 重新检索
- 保存结果
- 引用到最终提案

### 3.3 论文卡片字段

- 标题
- 作者
- 年份
- 期刊/会议
- 摘要
- AI 一句话贡献总结
- DOI 或外部链接
- 标签：review、empirical、theory、method、dataset
- 操作：收藏、引用、相关论文、加入提案

## 4. 技术架构

### 4.1 前端

推荐技术：

- React
- TypeScript
- Vite
- React Router
- Ant Design 或 shadcn/ui
- Zustand 或 Redux Toolkit
- CSS Modules 或 Tailwind CSS

关键前端模块：

- AgentChat：对话组件
- WorkflowStepper：工作流步骤
- PaperEvidencePanel：论文证据面板
- PaperCard：论文卡片
- SearchFilters：年份、类型、来源筛选
- ProposalEditor：提案草稿编辑区

### 4.2 后端

推荐技术：

- FastAPI 或 NestJS
- PostgreSQL
- Redis
- Celery/RQ 或 BullMQ
- Meilisearch / Typesense / Elasticsearch
- pgvector / Qdrant / Milvus

关键后端模块：

- 用户与会话管理
- 论文检索服务
- 论文元数据标准化
- Agent 工作流编排
- LLM 调用服务
- 向量检索服务
- 文献库与收藏服务
- 提案生成服务

### 4.3 数据源

优先接入：

- OpenAlex
- Semantic Scholar
- Crossref
- arXiv
- PubMed

后续可扩展：

- Google Scholar，需注意反爬和合规问题
- 中文论文数据库，需处理授权和数据来源合规
- 用户上传 PDF

## 5. Agent 工作流设计

### 5.1 检索增强生成流程

基本流程：

1. 用户输入研究方向
2. LLM 生成检索关键词和检索式
3. 后端调用论文数据源
4. 对论文去重、排序、过滤
5. LLM 基于论文证据总结贡献和挑战
6. 生成 idea
7. 用户追问或进入下一步验证

### 5.2 重要原则

- 所有学术判断尽量绑定文献证据
- AI 生成的 idea 必须标注不确定性
- 新颖性验证只给风险判断，不给绝对保证
- 用户可以查看每个结论对应的论文依据
- 系统需要记录每次检索式和筛选条件

## 7. MVP 开发里程碑

### Phase 1：静态原型

- 完成主页面布局
- 完成 6 步工作流 UI
- 完成论文卡片和证据面板
- 使用 mock 数据跑通完整流程

### Phase 2：论文检索

- 接入 OpenAlex 或 Semantic Scholar
- 支持关键词搜索
- 支持年份筛选
- 支持 review 关键词验证
- 保存检索结果

### Phase 3：Agent 总结

- 接入 LLM
- 实现研究空白扫描
- 实现 idea 生成
- 实现理论框架和方法缺口分析

### Phase 4：新颖性与可行性

- 实现 idea 新颖性验证
- 实现数据可行性分析
- 实现方法缺口分析
- 支持用户把结果加入提案上下文

### Phase 5：提案生成

- 生成 300 字研究提案
- 输出参考文献
- 支持保存和导出 Markdown

## 8. 第一版不做的事情

为了保证 MVP 可落地，第一版暂不做：

- 大规模 PDF 全文解析
- 自动爬取 Google Scholar
- 知网等商业数据库直连
- 复杂团队协作
- 引文网络可视化
- 端到端论文写作
- 绝对新颖性判定

## 9. 风险与注意事项

- 论文数据源覆盖不完整，可能漏掉重要文献
- LLM 可能产生幻觉，必须用文献证据约束输出
- 新颖性验证只能降低风险，不能证明无人研究过
- 学术建议不能替代导师判断
- 中文论文数据源和商业数据库需要特别注意版权和授权

## 10. 推荐默认提示词

### 研究空白扫描

```text
我是[领域]方向的硕/博士生，研究兴趣是[具体方向]。
请帮我列出该方向过去3年最重要的10篇论文及其核心贡献；
识别目前仍未解决的3个主要研究挑战；
基于上述挑战，提出2-3个可能的论文idea。
请为每个判断提供对应文献依据。
```

### 理论框架筛选

```text
围绕上一步生成的idea，请分析该议题最主流的3个理论框架。
请比较它们的解释对象、适用场景、局限性，以及对普通硕博生的可操作性。
```

### 新颖性验证

```text
请根据这个idea生成检索关键词和检索式。
分别搜索普通关键词、关键词+review、以及2025-2026年的最新论文。
判断这个方向是否已有综述或高度相似研究，并指出新颖性风险。
```

### 数据可行性分析

```text
这是我能获取的数据集：[数据集描述和变量列表]。
请判断样本覆盖是否匹配研究问题，变量是否足够支撑分析，
预期结论是否具有可证伪性，并建议可以反推的研究问题。
```

### 方法缺口分析

```text
请分析这个议题过去主要使用哪些研究方法。
哪种方法使用最多？局限在哪里？
是否有计算社会科学、机器学习文本分析、因果推断等新兴方法尚未广泛引入？
如果引入，能回答哪些过去答不了的问题？
```

### 研究提案生成

```text
请基于以上文献扫描、idea筛选、新颖性验证、数据可行性和方法缺口分析，
生成一段约300字的研究提案草稿。
内容包括研究背景、研究问题、文献空白、数据、方法和预期贡献。
```
